## 关键发现

### 1. GamePage 重定向到 /login ⚠️（预期行为）
- **原因**: `GamePage.tsx` 第117行：`if (!token) { navigate('/login') }`
- **判定**: 🔵 **非 bug** — 游戏页需要身份验证，这是有意的 auth guard
- **改进建议**: 如果不需要后端 API，可添加 guest 模式绕过

### 2. 不存在路由(`/nonexistent`) 渲染 TabBar ⚠️（低优先级）
- **现状**: 落在 `AppLayout/*` 通配路由内，显示 TabBar 但内容区空白
- **改进建议**: 可添加 `<Route path="*" element={<NotFound />} />` 给出 404 友好提示

### 3. 登录页 Form 验证 ✅
- 空表单提交 → `handleSubmit` 触发 `setError('请填写所有字段')`
- 用户名/密码均有 `minLength` 约束（UI 提示 "至少3位字符"）
- 注册模式有邮箱验证码 + 60s 倒计时

### 4. 配置页有 Advanced Settings 选项（检查了 Chrome 配置推送功能）
- 包含 Toggle 开关
- 已发现 `--remote-debugging-port` 配置，Chrome 启动参数可定制
